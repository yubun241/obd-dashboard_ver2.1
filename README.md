## Ver.3 主な進化

- **下半分パネルが全スロット選択式** — タップで切り替え
- **パネル数 0〜5 可変** — 等幅で自動分割
- **9種類のウィジェット** — RPM/速度/ギア/水温/油温/吸気温/ブースト/スロットル/Est.HP/Gメーター
- **G-METER** — スマホ加速度センサーで横G・縦Gを2D円表示、アラート閾値超えで赤発光
- **Est.HP** — MAP × RPM × 係数 で瞬間馬力を推定
- **OBD取得最適化** — 表示中ウィジェットが必要なPIDだけ取得、設定変更で自動再計算

## G-METER 仕様

### 座標系

- スマホを「立てた状態」（画面が垂直、内カメラ右）で使用想定
- 重力ベクトルをローパスフィルタで動的推定し、**自動で除去** → どんな姿勢でも静止時はセンター
- 円の中央 = 0G、端 = 設定値（例: 1.5G）

### 表示
- **横軸**: 旋回G (L/R)
- **縦軸**: 加減速G (A=加速 / B=ブレーキ)
- 数値表示: 横G / 縦G を G単位で

### アラート
- 合成G (`√(横²+縦²)`) が閾値を超えるとドット＆円が**赤く塗りつぶし発光**
- CFGで閾値を G単位で文字入力 (例: 1.0)

### 設定 (CFG)
- 表示範囲 `gMaxG` (G)
- アラート閾値 `gAlertG` (G)
- 軸反転 (横/縦個別)
- **キャリブレーション**: 「今この姿勢をセンター」ボタンで即時ゼロ点リセット

> ⚠ G-METERが想定と逆向きに動く場合は、CFG → 軸反転 で調整してください。

## Est.HP 仕様

- 計算式: `PS = MAP(kPa) × RPM × hpCoeff`
- デフォルト係数: `0.000197` (Mini F56 JCW 2.0L 基準: 231PS / 5200rpm / ~225kPa)
- 他車種は CFG で係数を調整
- バー表示の上限HPも設定可能

## OBD 取得最適化（Ver.3 新機能）

各ウィジェットは必要なPIDを宣言しており、**表示中のウィジェットだけポーリング**します。

### ポーリング戦略
- **高頻度 (毎サイクル)**: RPM (010C)、速度 (010D)
- **低頻度 (順番に1個ずつ)**: 表示中ウィジェットが要求するスロー系PID

### 動的再計算タイミング
- ウィジェット選択変更時
- パネル数変更時
- ブースト表示ON/OFF切替時
- 設定保存時 / 初期化時

→ 不要なPIDは取得しないので、**レスポンス速度が向上**します。

例: パネルを「SPEED + GEAR」だけにすれば 010C, 010D のみ高速取得。

## デプロイ手順 (GitHub Pages)

1. 新規リポジトリを作成
2. このディレクトリの内容をプッシュ
3. Settings → Pages → Branch: main / root を指定
4. 公開URLを Android Chrome で開く
5. ホーム画面に追加してフルスクリーン使用

### 更新時のキャッシュ対策

`sw.js` の `CACHE_VERSION` を必ず変更してください（例: `v3-001` → `v3-002`）。  
これで旧キャッシュが自動破棄され、新バージョンが反映されます。
